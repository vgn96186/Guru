# Plan: UI/UX Polish Analysis & Implementation

## Overview

This plan focuses on UI/UX polishes tailored to the app's ADHD medical student demographic, emphasizing immediate feedback, robust error prevention, and structured fallback states. This runs orthogonal to the ongoing NativeWind migration.

## Identified Polish Areas

### 1. The "Blank Screen" Fix (Empty States)

**Problem:** Several vault and bank screens render entirely blank when there are no items to display. This breaks the app's structured feel and fails to guide the user on what to do next.
**Affected Screens:**

- `NotesVaultScreen.tsx`
- `TranscriptVaultScreen.tsx`
- `RecordingVaultScreen.tsx`
- `QuestionBankScreen.tsx`
- `FlaggedContentScreen.tsx`
- `FlaggedReviewScreen.tsx` (Currently uses a custom view that bypasses the theme's `EmptyState` component).

**Solution:**

- Integrate the existing `src/components/primitives/EmptyState.tsx` into the `ListEmptyComponent` of the `FlatList` on these screens.
- Supply contextual `title` and `subtitle` props (e.g., "No recordings found", "Start a lecture to record audio.").

### 2. The "Obscured Content" Fix (Keyboard Handling)

**Problem:** Input-heavy screens lack a `KeyboardAvoidingView`. When a user types a long manual note or a daily log, the soft keyboard covers the text input and the "Save" button, leading to a frustrating experience.
**Affected Screens:**

- `ManualNoteCreationScreen.tsx`
- `ManualLogScreen.tsx`

**Solution:**

- Wrap the main `<ScrollView>` and `<View style={styles.footer}>` content with `react-native-keyboard-controller`'s `KeyboardAvoidingView` to ensure input fields remain visible above the keyboard.

### 3. The "Dopamine Hit" Fix (ADHD Micro-interactions)

**Problem:** While haptics are used well in many places, critical study actions like answering a core flashcard quiz or mastering a topic lack immediate, satisfying visual or haptic feedback.
**Affected Components:**

- `src/screens/ContentCard.tsx` (`QuizCard` component)
- `src/screens/TopicDetailScreen.tsx` (Mastered action)

**Solution:**

- **Haptics:** Add `Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success/Error)` to `QuizCard`'s `handleSelect` when answering questions.
- **Visual Motion:** Integrate `motion.pulseScale` or `motion.shake` from `src/motion/presets.ts` on quiz option buttons when selected, giving the UI a responsive, tactile feel.

## Next Steps

Once this plan is approved, we will iteratively tackle each of the three polish areas, verifying the UI on the Genymotion emulator after each change.
