# Implementation Plan: Redesign Settings into Dashboard UI

## Objective

Refactor the entire `SettingsScreen.tsx` and its child components into a modern, Linear-inspired dashboard UI as defined in `mockups/settings-dashboard.html`. The new design features a collapsible left sidebar for navigation and a responsive, symmetrical bento-box grid for the settings content.

## Architecture & Layout Changes

1. **Navigation Model**:

   - Replace the current 4-tab top-row system (`Account`, `Study`, `AI`, `Storage`) with a **7-category left sidebar**:
     1. General Overview
     2. AI & Inference
     3. Interventions
     4. App Integrations
     5. Planning & Alerts
     6. Device Sync
     7. Data & Storage
   - Implement a collapsible left sidebar that can be toggled via a hamburger menu in the top header.

2. **Main Layout (`SettingsScreen.tsx`)**:
   - Wrap the screen in a flex-row layout (Sidebar + Main Content).
   - **Header**: Contains the sidebar toggle, breadcrumbs, and the "Saving..." auto-save indicator.
   - **Quick Stats Banner**: Display the user's Total XP, Streak, and Topics Mastered prominently at the top of the "General Overview" section.
   - **Content Grid**: Use a responsive grid (1 column on mobile, 2 columns on tablet/desktop) with `bento-card` styling for the individual settings blocks.

## Component Refactoring Strategy

1. **Retain Core Logic**:

   - **DO NOT** alter the underlying database queries, OAuth handlers, permission requests, or the 600ms debounced auto-save architecture.
   - All state variables (`name`, `inicetDate`, `strictMode`, `chatgptAccounts`, etc.) remain identical to ensure no regressions in data saving.

2. **New Section Components**:
   Create/update components in `src/screens/settings/sections/` mapped to the new categories:

   - `GeneralOverviewSection.tsx`: Profile Name, Exam Dates.
   - `AiInferenceSection.tsx`: Local Model toggle, Cloud Routing Priority, Chat Models, API Keys.
   - `InterventionsSection.tsx`: Doomscroll Shield, Face Tracking (ML Kit), Punishment Mode (Strict Mode), Focus Subjects, Block Card Types.
   - `AppIntegrationsSection.tsx`: Permissions (Notifications, Mic, File Access, Overlay), DBMCI/BTR links, Google Drive sync.
   - `PlanningAlertsSection.tsx`: Wake-up time, Session Length, Goal, Pomodoro config.
   - `DeviceSyncSection.tsx`: Link Device button, Body Doubling toggles.
   - `DataStorageSection.tsx`: DB Export, Clear Cache, Reset Progress, Backup schedules.

3. **Styling Strategy (NativeWind)**:
   - Discard `StyleSheet.create` and `GlassSurface` in favor of NativeWind classes (`className="..."`).
   - Replicate the exact Linear-inspired color palette from the mockup:
     - Background: `bg-[#141517]`
     - Cards: `bg-[#1B1C1F]`
     - Borders: `border-[#292A2D]`
     - Primary Text: `text-[#E8E8E8]`
     - Secondary Text: `text-[#8A8F98]`
     - Accent: `bg-[#5E6AD2]` (Brand color)
   - Create a reusable `<BentoCard>` primitive in `src/components/settings/` to wrap settings blocks consistently.

## Execution Steps

1. **Step 1: Create Reusable UI Primitives**

   - Create `BentoCard.tsx` for the `.bento-card` styling.
   - Create `SidebarNavItem.tsx` for the left navigation links.
   - Create the `SettingsSidebar.tsx` component to encapsulate the collapsible drawer logic.

2. **Step 2: Build New Section Components**

   - Map the massive state block from `SettingsScreen.tsx` into props for the 7 new section files.
   - Rebuild the UI of each section using the new NativeWind layout, replacing `SettingsSectionAccordion` with `BentoCard` where appropriate (some deeper settings might still use accordions inside the cards).

3. **Step 3: Refactor `SettingsScreen.tsx`**

   - Implement the outer layout (`flex-row h-full bg-[#141517]`).
   - Add state for `isSidebarCollapsed` and `activeCategory`.
   - Update the render method to conditionally render the active section based on the sidebar selection.
   - Ensure the debounced auto-save hook and saving indicators remain intact and visible in the new top header.

4. **Step 4: Cleanup & Polish**
   - Remove dead code (old tab components).
   - Test responsive layout on tablet (`w-64` sidebar) vs mobile (hidden sidebar by default, slides out on toggle).
   - Ensure all toggles and inputs reflect their exact states from `useProfileQuery`.
