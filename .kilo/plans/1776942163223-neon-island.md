# Settings Refactor Execution Plan

## Objective

Enhance the layout, categorization, standardize primitives, and wire up hidden profile flags for the Guru Settings screen, strictly adhering to the no-DB-migration rule and targeting the Galaxy Tab S10+ and S23 Ultra layouts.

## 1. Create Layout Hook (`useSettingsLayout`)

- **File:** `src/hooks/useSettingsLayout.ts`
- **Logic:** Implement responsive breakpoints based on window width:
  - `phone`: `< 720px` (horizontal chip nav or accordion).
  - `tabletPortrait`: `720px - 1179px` (2-pane: collapsed sidebar + content).
  - `tabletLandscape`: `>= 1180px` (3-pane: full sidebar + TOC rail + detail pane).

## 2. Standardize UI Primitives

Extract shared atoms into `src/screens/settings/components/` by wrapping existing primitives (`LinearButton`, `LinearSurface`, `LinearText`), ensuring no raw `Pressable` or `View` components are used with custom styles.

- **`SettingsActionButton.tsx`**: Wraps `LinearButton` (replaces `testBtn`/`dangerBtn` patterns). Props: `variant`, `size`, `loading`, `onPress`.
- **`SettingsChip.tsx`**: Wraps `LinearSurface` or `LinearChipButton` (replaces `typeChip`/`freqChip`). Props: `label`, `selected`, `onPress`, `color`.
- **`SettingsChipGrid.tsx`**: A responsive flex-wrap container for rendering multiple `SettingsChip` components.

**Theme Enforcement:**

- Update `src/screens/settings/components/SettingsToggleRow.tsx` to replace hardcoded hex values with `linearTheme.colors.*`.
- Update `src/components/settings/BentoCard.tsx` to use `linearTheme.colors.border` and derived gradient stops.
- Update `src/components/settings/SettingsSidebar.tsx` and `SidebarNavItem.tsx` to use `LinearText` variants instead of raw `Text`.

## 3. Create Adaptive Container (`SettingsSubSection`)

- **File:** `src/screens/settings/components/SettingsSubSection.tsx`
- **Logic:** Uses `useSettingsLayout()`. On `tabletLandscape`, it renders content inside a `BentoCard` (flat, visible). On `phone` and `tabletPortrait`, it renders as a collapsible accordion (e.g., `SettingsSectionAccordion`).

## 4. Reorganize Settings Categories (9 Total)

Update `src/screens/SettingsScreen.tsx` and `src/components/settings/SettingsSidebar.tsx` to implement 9 categories with 2-level grouping:

1. **General**: Profile identity, device link, level/xp glance.
2. **Appearance & Accessibility** _(New)_: `loadingOrbStyle`, `orbEffect`, `useSPenControls`, `visualTimersEnabled`, `focusAudioEnabled`.
3. **Goals & Timeline (Planning)**: Dates (INICET/NEET, DBMCI/BTR) and Pace & Reminders (`preferredSessionLength`, `dailyGoalMinutes`, `studyResourceMode`, `notificationHour`).
4. **Focus & Rules (Interventions)**: Guardrails (`strictModeEnabled`, `doomscrollShieldEnabled`, `faceTrackingEnabled`, `skipBreaks`) and Session Rules (`idleTimeoutMinutes`, `breakDurationMinutes`, `pomodoroEnabled`).
5. **AI Intelligence**: Models & Memory, OAuth Connections, Keys & Routing, Local AI (`useNano`), Image & Transcription (`huggingFaceTranscriptionModel`).
6. **Integrations & OS**: Permissions and Reliability (Samsung background whitelist).
7. **Device Sync**: Presence (`bodyDoublingEnabled`) and Google Drive (moved from Storage).
8. **Data & Storage**: Backup and Maintenance (`autoRepairLegacyNotesEnabled`, `scanOrphanedTranscriptsEnabled`).
9. **About & Diagnostics** _(New)_: App version, logs, and permission audit.

_Note:_ Update status dots in the sidebar using existing computed counts (`providerReadyCount`, etc.).

## 5. Implement & Update Sections

Wire all ~12 hidden profile fields end-to-end using the existing `useSettingsState(key, default)` hook.

- **`DashboardOverview.tsx` fixes:**
  - Wire `doomscrollShieldEnabled` via `useSettingsState`.
  - Format `profile.notificationHour` instead of hardcoded "06:30 AM".
  - Compute local model badge from `localLlmReady`.
  - Calculate progress bar using today's minutes vs `dailyGoalMinutes`.
  - Bind DBMCI/BTR cards to `dbmciClassStartDate` / `btrStartDate`.
- **Section Refactors:**
  - Shrink `GeneralOverviewSection.tsx` to identity only.
  - Create `AppearanceSection.tsx` and `AboutDiagnosticsSection.tsx`.
  - Update `InterventionsSection.tsx`, `PlanningAlertsSection.tsx`, `DeviceSyncSection.tsx`, and `DataStorageSection.tsx` to match the new groupings.
  - Remove DBMCI/BTR from `AppIntegrationsSection.tsx`.
  - Add `useNano` to `LocalAiSection.tsx` and `huggingFaceTranscriptionModel` to `TranscriptionSection.tsx`.

## 6. Clean Up Legacy Components

Delete the following unused components (and their tests) from `src/components/settings/`:

- `NotificationSection.tsx`
- `StudyGoalsSection.tsx`
- `StudyPreferencesSection.tsx`
- `ContentPreferencesSection.tsx`
- `AdvancedToolsSection.tsx`
- `ApiKeySection.tsx`
- `PermissionRow.tsx`
- `ProfileSection.tsx` (Move `TextField.tsx` to `src/screens/settings/components/` if still needed).

## 7. Strict Constraints & Quality Assurance

- **Imports:** Strictly use relative paths (e.g., `../../components/...`) within `src/`. Do NOT use `@/`.
- **State Management:** Rely solely on `useSettingsState` for persisting toggles (it automatically triggers `refreshProfile()`).
- **No DB Migrations:** Only surface fields already existing in `UserProfile`.
- **Testing:** Run `npm run verify:ci` to ensure lint and tests pass. Update or remove legacy tests associated with deleted components.
